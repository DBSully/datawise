# REcolorado Basic 50 Import Mapping

Import profile: `recolorado_basic_50`  
Source system anchor: `recolorado`  
Default state rule: `CO`

## Notes

- `source_system = 'recolorado'` should be recorded on source-derived tables (`mls_listings`, `property_financials`, `import_batches`, `import_batch_rows`).
- `state = 'CO'` should be applied during import because the file itself does not contain a State column.
- `normalized_address_key`, `address_slug`, `public_code`, `id`, `created_at`, and `updated_at` are generated by DataWise and are not expected in the CSV.
- `property_type` and `level_class_standardized` are DataWise-standardized outputs, not raw MLS fields.

## DataWise-generated fields

### real_properties

- `id`
- `public_code`
- `state = 'CO'`
- `normalized_address_key`
- `address_slug`
- `geocode_source = 'mls'` when latitude/longitude present
- `created_at`
- `updated_at`

### property_physical

- `real_property_id`
- `property_type` (generated from subtype/structure rules)
- `level_class_standardized` (generated from levels rules)
- `created_at`
- `updated_at`

### mls_listings

- `id`
- `real_property_id` (after property match/upsert)
- `source_system = 'recolorado'`
- `created_at`
- `updated_at`

### property_financials

- `real_property_id`
- `source_system = 'recolorado'`
- `source_record_id = listing_id`
- `created_at`
- `updated_at`
